## âœ… Apa Saja yang Saya Lakukan?
Dalam project ini, saya tidak hanya mengecek "Flow Positif" (Happy Path), tapi juga mencari celah keamanan melalui "Flow Negatif".

1.  **CRUD Testing**: Memastikan fungsi Create, Read, Update, dan Delete berjalan lancar.
2.  **Validasi Data**: Memastikan tipe data benar (misal: ID harus angka).
3.  **Negative Testing**: Sengaja memasukkan data salah untuk melihat respon server.
4.  **Bug Hunting**: Mencari celah error dan keamanan pada API.
